API Help

This website only contains the API features currently. We're working on the rest :) To use the API and explore how things work try using some of the following links

GET Information

GET requests are requests to websites for information, they are the most common request made. You can make a GET request by simply pointing your browser to the information you wish to GET.

  • List the entities
    • /api/contact/list/
    • /api/department/list/
    • /api/equipment/list/
    • /api/institution/list/
    • /api/keyword/list/
    • /api/protocol/list/
  • Search for entities, the q=a part in these links means search query = 'a', change the 'a' to change the search
    • /api/contact/search/?q=a
    • /api/department/search/?q=a
    • /api/equipment/search/?q=a
    • /api/institution/search/?q=a
    • /api/keyword/search/?q=a
    • /api/protocol/search/?q=a
  • Lookup an individual entity, the number at the end of the link is the entity's ID, find IDs by listing or searching
    • /api/contact/1/
    • /api/department/1/
    • /api/equipment/1/
    • /api/institution/1/
    • /api/keyword/1/
    • /api/protocol/1/

POST New Information

A POST request is used when you wish to create a new entity, to add something to the database. To POST something you'll need some software that allows POSTing of data (There's a Firefox plugin called Poster, for instance).

  • Create an entity. POST the JSON data to the URL using Content-Type: application/json
    • URL: /api/contact/, Data: {"name":"Toby", "email":"toby@iiiscience.com"}
    • URL: /api/department/, Data: {"name":"Department of Materials", "institution":1} (institution must take a valid institution ID)
    • URL: /api/equipment/, Data: {"name":"SEM", "details":"details", "department":1, "contact":1, "keywords":["keyword", 1]} (keywords can contain new keywords or keyword IDs)
    • URL: /api/institution/, Data: {"name":"University College London"}
    • URL: /api/keyword/, Data: {"keyword":"beta"}
    • URL: /api/protocol/, Data: {"name":"SEM", "details":"details", "url":"http://example.com", "department":1, "contact":1, "keywords":["keyword"]}

PUT Updated Information

A PUT request is used when an entity already exists but you'd like to update the information. To PUT something you'll need some software that allows PUTting of data (There's a Firefox plugin called Poster, for instance).

  • Update an entity. PUT the JSON data to the URL (containing the ID) using Content-Type: application/json
    • URL: /api/contact/1/, Data: {"name":"Toby", "email":"toby@iiiscience.com"}
    • URL: /api/department/1/, Data: {"name":"Department of Materials", "institution":1} (institution must take a valid institution ID)
    • URL: /api/equipment/1/, Data: {"name":"SEM", "details":"details", "department":1, "contact":1, "keywords":["keyword", 1]} (keywords can contain new keywords or keyword IDs)
    • URL: /api/institution/1/, Data: {"name":"University College London"}
    • URL: /api/keyword/1/, Data: {"keyword":"beta"}
    • URL: /api/protocol/1/, Data: {"name":"SEM", "details":"details", "url":"http://example.com", "department":1, "contact":1, "keywords":["keyword"]}
